# 프레젠테이션 구현 명세

## 기본 원칙

- 이 프로젝트는 브라우저 기반 16:9 프레젠테이션이다.
- 발표 대본이 슬라이드 구현의 source of truth이다.
- 내부 클릭은 대본의 `[클릭]` 위치에만 존재한다.
- 슬라이드 전환 입력은 대본의 `[클릭]`에 포함하지 않는다.
- 미세 애니메이션마다 별도의 Step을 만들지 않는다.
- 한 수동 Beat 안에서 여러 요소가 자동으로 순차 연출될 수 있다.
- 일반 슬라이드는 내부 클릭 0~1회이다.
- 핵심 반전 슬라이드만 최대 2회이다.
- 누적형 HUD 화면을 피하고, 한 장마다 중심 시각 요소 하나를 둔다.
- 작은 영문 라벨과 장식용 시스템 텍스트를 남발하지 않는다.
- 실제 브라우저 슬라이드가 디자인의 source of truth이다.
- 소셜 미리보기 이미지가 실제 슬라이드보다 더 완성도 높거나 다른 디자인이 되면 안 된다.
- 별도의 대본 웹페이지나 presenter view는 만들지 않는다.
- 기존 커스텀 React 프레젠테이션 엔진은 유지한다.

## Beat 동작 규칙

- 한 번의 수동 입력은 발표자가 다음 설명 문단으로 넘어가는 한 지점만 처리한다.
- 같은 Beat의 경로 표시, 비트 점등, 숫자 변경, 결론 등장은 자동 sequence로 처리한다.
- 뒤로 이동하면 이전 Beat의 장면 상태를 정확히 복원한다.
- 자동 sequence는 Beat 상태에 따른 CSS animation과 delay로 재생하며, 중간 상태를 위한 React Step을 추가하지 않는다.
- 길게 누른 키의 반복 입력은 무시한다.

## 전체 내부 클릭 예산

| 슬라이드 | 내부 클릭 |
| --- | ---: |
| Slide 1 | 0 |
| Slide 2 | 1 |
| Slide 3 | 1 |
| Slide 4 | 1 |
| Slide 5 | 1 |
| Slide 6 | 2 |
| Slide 7 | 1 |
| Slide 8 | 1 |
| Slide 9 | 1 |
| Slide 10 | 0 |
| **합계** | **9** |

현재 실제 구현 범위는 Slide 1~6이다. Slide 7~10의 화면은 아직 구현하지 않는다.
